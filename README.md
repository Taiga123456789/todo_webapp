# Todoリスト Webアプリケーション  
（オブジェクト指向プログラミング 最終課題）

本リポジトリは、情報工学科3学年前期「オブジェクト指向プログラミング」の最終課題として開発した  
Todoリスト管理用Webアプリケーションです。

Webアプリケーションの基本構成（フロントエンド・バックエンド・データベース）を理解し、  
クライアントとサーバ間の通信やCRUD処理を自力で実装することを目的として開発しました。

---

## 概要

本アプリケーションは、ブラウザ上からタスクの登録・閲覧・更新・削除を行える  
シンプルなTodo管理Webアプリです。

フレームワークは使用せず、HTML / JavaScript / PHP / SQLite を用いて  
Webアプリケーションの基礎構造を一から実装しています。

---

## 実装機能

- タスクの新規登録（Create）
- タスク一覧の取得・表示（Read）
- タスク完了状態の更新（Update）
- タスクの削除（Delete）
- Fetch API を用いた非同期通信（ページ遷移なし）

---

## 使用技術

### フロントエンド
- HTML
- CSS（Tailwind CSS）
- JavaScript
- Fetch API（非同期通信）

### バックエンド
- PHP
- REST風API設計

### データベース
- SQLite
- PDO（PHP Data Objects）

---

## システム構成
- フロントエンドはUI表示およびユーザー操作を担当  
- バックエンドはAPIとしてCRUD処理を提供  
- データはSQLiteに永続化  

---

## API設計

バックエンドではHTTPメソッドごとに処理を分岐し、  
CRUD操作を明確に分離しています。

| HTTPメソッド | エンドポイント | 内容 |
|-------------|----------------|------|
| GET | api.php | タスク一覧の取得 |
| POST | api.php | タスクの追加 |
| PATCH | api.php | タスク状態の更新 |
| DELETE | api.php | タスクの削除 |

HTTPメソッドを明示的に使い分けることで、  
処理内容の可読性と保守性を意識しました。

---

## オブジェクト指向を意識した点

本課題では、以下の点でオブジェクト指向の考え方を意識しました。

- フロントエンド・バックエンド・DBの責務分離
- PDOを用いたデータアクセスの抽象化
- CRUD操作をHTTPメソッド単位で整理
- 処理の役割が分かりやすいコード構成

---

## 工夫した点・学んだこと

- Fetch API を用いた非同期通信により、UXを向上させた  
- プリペアドステートメントを使用し、SQLインジェクション対策を実施  
- Webアプリケーションの基本的な通信フローを実装レベルで理解できた  
- フレームワークを使わずに実装することで、Webの基礎構造を深く理解できた  

---

## 起動方法

1. `index.html` と `api.php` を同一ディレクトリに配置
2. PHPが動作するWebサーバー（例：XAMPP）を起動
3. ブラウザから `index.html` にアクセス
